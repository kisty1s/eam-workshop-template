This worklog summarizes the activities completed during the 12-week internship period at Amazon Web Services Vietnam. The weekly records show the learning path from foundational AWS services to cloud infrastructure design, backend development, system deployment, documentation, testing, and final project preparation.

Each week focuses on a specific technical area and connects directly to the Enterprise Asset Management (EAM Workspace) project and the First Cloud AI Journey training roadmap.

## Weekly Summary

**Week 1:** [Getting familiar with AWS, cloud computing fundamentals, and the internship environment](1.1-week1/)

**Week 2:** [Learning Amazon VPC, network layers, EC2 deployment, and AWS support credit setup](1.2-week2/)

**Week 3:** [Practicing Amazon S3 storage, bucket structure, IAM users, groups, and permissions](1.3-week3/)

**Week 4:** [Setting up Linux environments, EC2 system administration, web servers, and runtime configuration](1.4-week4/)

**Week 5:** [Researching and configuring ELB, Auto Scaling Groups, high availability, and performance testing](1.5-week5/)

**Week 6:** [Learning Docker, writing Dockerfiles, building container images, and using AWS CLI](1.6-week6/)

**Week 7:** [Developing backend CRUD APIs, asset filtering logic, and studying advanced AWS networking](1.7-week7/)

**Week 8:** [Implementing enterprise features including e-signature workflows, CSAT forms, reporting, AI support, and live chat](1.8-week8/)

**Week 9:** [Designing system architecture, AWS deployment layouts, and technical blueprints](1.9-week9/)

**Week 10:** [Finalizing system diagrams, standardizing technical documentation, and aligning project architecture](1.10-week10/)

**Week 11:** [Conducting end-to-end testing, code review, backend optimization, and system validation](1.11-week11/)

**Week 12:** [Completing the internship report, technical manuals, final presentation, live demo, and project handover](1.12-week12/)
