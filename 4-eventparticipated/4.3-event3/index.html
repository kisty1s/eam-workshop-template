Event 3: Study Tour: AWS - Enterprise Cloud Architectures & Industry Applications

Event Information

ItemDetails
DateJuly 4, 2026
LocationAWS Vietnam Office, Bitexco Financial Tower
Participation formatIn-person participation
RoleAttendee
Main topicsEnterprise Cloud Architectures, GenAI, Data Engineering, career development roadmap, and practical enterprise technology applications

Overview

Study Tour: Amazon Web Services (AWS) was an in-depth practical technology experience jointly organized by Swinburne Vietnam and AWS Vietnam. The event brought together more than 100 outstanding technology students who traveled more than 200 km from early morning to connect directly with and learn from industry experts.

The core objective of the Study Tour was to examine how large enterprises apply Cloud Computing and Generative AI (GenAI) to operate large-scale data systems, while shaping a practical architecture mindset for the next generation of engineers.

Key Topics

Overview of the Cloud and GenAI Market

  • Technology shift: The session analyzed how Cloud Computing and GenAI are reshaping the operating models of the modern technology world. Enterprises no longer treat cloud only as storage infrastructure, but as a core platform for deploying reasoning models and processing real-time Big Data.
  • Recruitment trends: Speakers updated students on new hiring criteria from global technology enterprises. The market is moving from looking only for people who can code toward engineers with system thinking, Cloud Architecture knowledge, and the ability to use GenAI to improve work performance.

Enterprise Data Architecture: From Classroom to Practice

  • Data optimization problems: The event shared practical stories about how large enterprises design, operate, and optimize massive Data Architectures in real environments to ensure high availability, security, and scalability.
  • Capability transition: The session described the path from being a theory-focused student to becoming a production-ready Data Engineer. It emphasized real constraints such as system latency, infrastructure cost optimization, and centralized data governance.

Closing the Gap Between Learning and Practice

  • How to stand out to employers: Speakers guided students on turning theoretical knowledge and university projects into practical Solutions Architecture that can run in real enterprise contexts.
  • Approach methodology: Students were encouraged to approach technology from a business-driven perspective instead of only following technology trends. This is a key mindset for creating technology products with real commercial value.

Fireside Chat: Nurturing Breakthrough Technology Talent

  • Expert discussion: The roundtable brought together representatives from AWS Vietnam, Renova Cloud, Cloud Kinetics, and outstanding GenAI engineers to discuss how to nurture and develop the next generation of breakthrough technology talent in Vietnam.
  • Leading mindset: The discussion reinforced that technology may change and become outdated every day, but problem-solving mindset and a proactive practical spirit are the core capabilities that help young engineers avoid being replaced by AI and stay ahead of new technology trends.

Key Takeaways

  • The intersection of Cloud and AI: GenAI needs powerful and flexible cloud infrastructure to scale up; in return, cloud needs GenAI to automate and make large-scale operations more intelligent.
  • The value of practical thinking: Academic knowledge is only the necessary foundation. Mastering tools, understanding system architecture, and handling incidents in real environments are the sufficient conditions for becoming an enterprise engineer.
  • Security and data governance: When operating massive enterprise data systems, data security layers and cost optimization (FinOps) must be considered from the first architecture design stage.
  • Adaptability: Self-learning and proactive technology updates are mandatory in an era where cloud and AI services improve every week.

Connection to the EAM Workspace Project

The AWS Vietnam Study Tour provided valuable architecture lessons that can be applied directly to improve the EAM Workspace project, an enterprise asset management system:

  • Cloud-native architecture design: The project should move toward packaging the system with containerization standards such as Docker and Kubernetes, making it ready to migrate and deploy flexibly to cloud services such as AWS ECS or EKS. This helps the system support smooth auto-scaling when asset data and employee access increase suddenly.
  • Data pipeline optimization: Practical Data Engineering thinking can be applied to standardize the asset management database structure, optimize queries, and establish real-time data synchronization between branches and departments without overloading the backend.
  • GenAI integration: The project can explore secure API-based GenAI model integration to build intelligent asset data analysis, automatic depreciation forecasting, equipment lifecycle alerts, or natural-language chat for asset inventory reports.

Conclusion

The AWS Study Tour provided a close view of how large-scale enterprises operate systems in real environments. The event reinforced that a successful enterprise software system such as EAM Workspace should not only run the required features correctly, but also be built on a well-designed cloud architecture that is secure, highly scalable, and ready to integrate breakthrough technologies such as AI to optimize business processes.
